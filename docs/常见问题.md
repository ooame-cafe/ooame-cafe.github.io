# 常见问题

## 双击工具打不开

第一次运行时，右键 `.command` 文件，选择“打开”。如果提示没有执行权限，在博客目录运行：

```bash
chmod +x *.command bin/*.sh
```

## `node` 或 `npm` 未找到

安装 Node.js LTS，关闭并重新打开终端，再执行：

```bash
node --version
npm --version
```

Hexo 8 需要 Node.js 20.19 或更高版本。

## 本地后台没有“使用本地仓库”

确认：

1. 使用 Chrome、Edge 等 Chromium 浏览器；
2. 地址为 `http://localhost:4000/admin/index.html`；
3. 选择的是包含 `_config.yml` 和 `.git` 的博客根文件夹。

Safari 和 Firefox 当前不支持 Sveltia 所需的本地文件访问接口。

## 公网后台要求令牌

这是个人 GitHub 后台的登录方式。使用后台提供的链接创建令牌，不要把令牌粘贴到 `_config.yml`、文章或 GitHub 仓库中。

如果更换设备，需要在新设备重新登录。设备不再使用时，应退出后台并在 GitHub 撤销对应令牌。

## 图片上传后不显示

先检查图片是否进入 `source/uploads/`，再运行：

```bash
npm run build
```

生成后的图片应位于 `public/uploads/`。本模板按自定义域名根路径设计，`public_folder` 固定为 `/uploads`。

## GitHub Actions 构建失败

常见原因：

- `_config.yml` 仍有 `example.com` 或 `your-github-username`；
- YAML 缩进被破坏；
- `package.json` 与 `package-lock.json` 不一致；
- 文章头部的 YAML 格式不完整。

本地先运行：

```bash
npm ci
npm run build
```

本地错误信息通常更容易阅读。

## 域名打开是 404

依次检查：

1. Actions 是否已经成功；
2. `Settings → Pages → Source` 是否为 GitHub Actions；
3. `Custom domain` 是否与 `_config.yml` 一致；
4. 子域名 CNAME 是否指向 `用户名.github.io`，且没有仓库名；
5. DNS 是否仍在传播。

## 同步发布出现冲突

脚本会停止并显示 Git 提示，不会静默忽略。先保留终端信息，不要删除 `.git`。可以使用 GitHub Desktop 查看冲突；不确定时先复制一份项目文件夹作为备份，再处理同一篇文章的两份改动。

## 搜索没有新文章

搜索索引在构建时生成。确认文章已经开启“公开发布”，再运行：

```bash
npm run build
```

本地刷新页面；公开站点需等待 Actions 完成。
