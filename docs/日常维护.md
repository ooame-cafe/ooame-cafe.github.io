# 日常维护

## 推荐顺序

在 Mac 本地写作时：

1. 双击 `检查更新.command`；
2. 双击 `开始写作.command`；
3. 在 Chrome 后台编辑并保存；
4. 在本地首页检查文章、图片和目录；
5. 双击 `同步发布.command`；
6. 需要额外保险时双击 `本地备份.command`。

从公网后台写作时，发布会直接进入 GitHub，不需要再从 Mac 推送。下次回到 Mac，先运行“检查更新”。

## 草稿

后台的“公开发布”开关关闭时，对应文章头部的：

```yaml
published: false
```

草稿会保存在仓库，便于跨设备继续编辑，但不会出现在首页、归档、分类、标签、搜索和 RSS 中。准备公开时关闭该开关并再次发布。

## 修改已有文章

在后台打开文章后直接修改。不要同时在两台设备编辑同一篇文章；如果确实需要切换设备，应先等待上一次 GitHub Actions 完成，再重新打开后台。

## 删除文章或图片

删除文章前先确认不再需要其中内容。文章删除后，曾经被其他文章引用的图片不会自动删除，需要在 `source/uploads/` 中单独检查。

本地误删且尚未同步时，可以用 Git 客户端恢复。已经推送到 GitHub 的内容可以从仓库历史中找回。

## 备份

`本地备份.command` 默认把 ZIP 放到项目旁边的 `shiso-blog-backups`。备份包含文章、图片、主题、配置和教程，不包含：

- `node_modules`；
- `public`；
- `.git`；
- 临时日志。

恢复时解压备份，运行 `npm install`，再连接 GitHub 仓库。

如需 Gitee 远程备份：

```bash
git remote add gitee https://gitee.com/你的用户名/你的备份仓库.git
npm run sync:all -- "更新博客"
```

未配置 `gitee` 远程时，`sync:all` 会明确提示并跳过。

## 更新依赖

不建议频繁追逐版本。需要更新时：

```bash
npm outdated
npm update
npm run build
```

构建和浏览器检查通过后再提交 `package.json` 与 `package-lock.json`。Sveltia CMS 当前固定版本，升级前应先备份并测试本地、远程后台。

## 图片控制

Git 不适合持续存放大量原始照片。建议在上传前：

- 把长边缩放到 2000 像素以内；
- 照片优先用 WebP 或质量适中的 JPEG；
- 截图可用压缩后的 PNG 或 WebP；
- 删除 EXIF 中不希望公开的位置、设备等信息。
