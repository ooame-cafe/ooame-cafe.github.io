# 从空仓库到公开博客：完整教程

本文按实际操作顺序说明如何建立、发布和维护博客。第一次配置建议预留约半小时，其中域名解析的生效时间由域名服务商决定。

## 一、准备账户和软件

需要：

1. 一个 GitHub 账户；
2. 自定义域名为可选项；没有域名时可以使用 GitHub 免费地址；
3. macOS 电脑；
4. Node.js 20.19 或更高版本；
5. Git；
6. Chrome 或 Edge，用于本地网页后台。

安装 Node.js 后，打开“终端”，检查：

```bash
node --version
npm --version
git --version
```

三条命令都能显示版本号再继续。

## 二、填写博客信息

双击根目录的 `首次设置.command`，依次填写：

- 博客名称；
- 作者名称；
- 副标题；
- 首页简介；
- 自定义域名；没有时输入 `-`；
- GitHub 用户名；
- GitHub 仓库名；
- 公开邮箱。

输入域名时可以写 `blog.example.com`，也可以粘贴 `https://blog.example.com`，向导会去掉协议和路径。公开邮箱可以留空。

所有结果保存在 `_config.yml`。该文件是公开配置，会上传到 GitHub，不要在其中填写密码、访问令牌或其他密钥。

## 三、创建 GitHub 仓库

登录 GitHub，右上角点击 `+ → New repository`：

1. 没有自定义域名时，`Repository name` 使用 `账户名.github.io`；
2. 选择 `Public`；GitHub Free 的 Pages 应使用公开仓库；
3. 不勾选 README、`.gitignore` 和 License；
4. 点击 `Create repository`。

保持新仓库页面打开，稍后需要复制仓库地址。

## 四、第一次上传

在 Finder 中打开博客文件夹，右键文件夹，选择“新建位于文件夹位置的终端窗口”。如果没有该菜单，也可以把文件夹拖到终端窗口中，再手动执行 `cd`。

执行：

```bash
git init -b main
git add .
git commit -m "创建个人博客"
git remote add origin https://github.com/你的GitHub用户名/你的仓库名.git
git push -u origin main
```

把最后两行的用户名和仓库名替换成自己的信息。若系统要求登录，按浏览器提示完成 GitHub 授权。

如果 `git commit` 提示没有姓名和邮箱，先执行：

```bash
git config --global user.name "你的公开名称"
git config --global user.email "你的GitHub邮箱"
```

然后重新执行 `git commit` 和后续命令。

## 五、开启 GitHub Pages

进入 GitHub 仓库：

```text
Settings → Pages
```

在 `Build and deployment` 下，把 `Source` 改为 `GitHub Actions`。之后进入 `Actions` 页面查看“构建并发布博客”。

如果任务失败：

1. 打开失败任务；
2. 展开“检查配置并构建”；
3. 按中文错误提示检查 `_config.yml`；
4. 修改后再次双击 `同步发布.command`。

## 六、设置自定义域名

建议先在 GitHub 账户设置中验证域名，再进入仓库 `Settings → Pages` 填写 `Custom domain`。保存成功后才去域名服务商修改 DNS。

### 子域名示例

目标地址为 `blog.example.com`：

```text
记录类型：CNAME
主机记录：blog
记录值：你的GitHub用户名.github.io
```

### 根域名示例

目标地址为 `example.com`，添加四条 `A` 记录：

```text
@ → 185.199.108.153
@ → 185.199.109.153
@ → 185.199.110.153
@ → 185.199.111.153
```

不要使用 `*.example.com` 这类通配 DNS。解析生效后回到 Pages 设置，勾选 `Enforce HTTPS`。

## 七、第一次本地写作

双击 `开始写作.command`。Chrome 打开后台后：

1. 选择“使用本地仓库”；
2. 选择博客根文件夹；
3. 打开“文章”；
4. 新建文章；
5. 填写标题、日期、分类、标签、摘要；
6. 可上传封面图，并在正文中继续插图；
7. 需要暂存时关闭“公开发布”；
8. 保存。

回到本地首页刷新，检查排版。确认无误后双击 `同步发布.command`。

## 八、第一次公网写作

访问：

```text
https://你的域名/admin/index.html
```

选择令牌登录。按后台链接到 GitHub 创建令牌，复制并粘贴。令牌只应保存在私人设备。

公网后台点击发布后会直接修改 GitHub 仓库。通常等待一两分钟，Actions 完成后网站就会更新。若同时在本地和公网编辑同一篇文章，应先在本地双击 `检查更新.command`，减少版本冲突。

## 九、修改外观和内容结构

常用内容都在 `_config.yml`：

```yaml
profile:
  eyebrow: PERSONAL NOTES
  intro: 在这里整理研究、阅读、项目和生活中的具体问题。
  footer: 写作是为了留下可复查的线索。
```

分类示例：

```yaml
content:
  categories:
    - 日常记录
    - 研究笔记
    - 读书
    - 项目
    - 旅行
```

改动后执行：

```bash
npm run build
```

构建通过再同步发布。
